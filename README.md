# James Perenchio — portfolio & work record

A multi-page, self-updating documentation site for everything I've been
building: my public software projects (with their real git histories) and the
self-hosted infrastructure I run.

**Live site:** https://jamesperenchio1.github.io/portfolio

## How it works

The site is plain static HTML/CSS — no framework, no client-side build. Pages
are generated by a single Node script that reads live data from the GitHub API:

```
node scripts/build.mjs
```

It produces:

| Page | Source |
|------|--------|
| `index.html` | GitHub repo list + commit counts |
| `projects.html` + `projects/*.html` | every public repo, with full commit history |
| `timeline.html` | unified chronological commit feed across all repos |
| `infrastructure.html` | rendered from `COMPREHENSIVE.md` |
| `log.html` | rendered from `PROJECT_LOG.md` |
| `about.html` | rendered from `content/about.md` |

## Staying up to date automatically

`.github/workflows/update-site.yml` runs the generator **every day** (and on any
source change), regenerates all pages from the latest GitHub data, and commits
the result. GitHub Pages then republishes. There is no manual update step.

To edit the hand-written pages, change the Markdown files (`content/about.md`,
`COMPREHENSIVE.md`, `PROJECT_LOG.md`) — the next build folds them in.

## Local build

```
node scripts/build.mjs            # unauthenticated (low API rate limit)
GITHUB_TOKEN=ghp_xxx node scripts/build.mjs   # authenticated (recommended)
```

Requires Node 18+ (uses the built-in `fetch`). No dependencies to install.
