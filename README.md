# James Perenchio — portfolio

A modern, self-updating portfolio for my public software projects, generated
straight from GitHub (with real commit histories) and refreshed on a schedule.

**Live site:** https://jamesperenchio1.github.io/portfolio

## How it works

The site is plain static HTML/CSS with a sprinkle of vanilla JS for the
interactions (theme toggle, scroll reveals, project search/filter) — no
framework, no client-side build. Every page is generated by a single Node
script that reads live data from the GitHub API:

```
node scripts/build.mjs
```

It produces:

| Page | Source |
|------|--------|
| `index.html` | GitHub repo list, commit totals, recently active projects |
| `projects.html` + `projects/*.html` | every public repo, with full commit history |
| `timeline.html` | unified chronological commit feed across all repos |
| `about.html` | rendered from `content/about.md` |

Styling lives in `assets/style.css` and the interactions in `assets/app.js`.

## Staying up to date automatically

`.github/workflows/update-site.yml` runs the generator on a schedule (and on any
source change), regenerates all pages from the latest GitHub data, commits the
result, and publishes to GitHub Pages. There is no manual update step.

To edit the hand-written About copy, change `content/about.md` — the next build
folds it in.

## Local build

```
node scripts/build.mjs                         # unauthenticated (low API rate limit)
GITHUB_TOKEN=ghp_xxx node scripts/build.mjs    # authenticated (recommended)
```

When the API is unavailable or rate-limited, the build falls back to
`data/seed.json`, which carries the last-known repo list and commit history so
offline builds stay complete. Requires Node 18+ (uses the built-in `fetch`); no
dependencies to install.
