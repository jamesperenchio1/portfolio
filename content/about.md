# About

I'm James Perenchio, a software developer. I like building useful things on the
web and shipping them — full applications, small data tools, and the automation
that ties them together.

This site is a single, honest place to see what I actually work on. Instead of a
hand-written résumé that goes stale, it reads my work directly from GitHub and
rebuilds itself on a schedule, so the project list, descriptions, and commit
history you see here stay close to live.

## What I work on

- **Web apps and storefronts** — building and iterating on full web applications
  and e-commerce, from the front end through to payments and checkout.
- **Data tools and trackers** — small, focused apps that collect real-world
  signals and present them clearly.
- **Automation** — pipelines that research, generate, and commit their own
  output on a schedule. A few of these repositories quite literally run
  themselves, and this website is built the same way.
- **Self-hosting** — running my own services on a small home server instead of
  renting everything from the cloud. The [Infrastructure](infrastructure.html)
  page documents that setup (with credentials and personal details redacted).

## How I work

I lean on automation wherever it removes busywork, I prefer shipping small and
often over big-bang launches, and I like keeping a clear record of what changed
and why. The commit histories linked throughout this site are that record.

## How this site works

There's no manual update step. A scheduled job re-reads my public GitHub data,
regenerates every page, and republishes. The generator that does it is itself one
of the public repositories listed under [Projects](projects.html).
